## 1. 系统概览
Alchemy-RSS 是一个高度自动化的信息提炼管道。它通过“配置驱动抓取 -> 逻辑层清洗 -> 资产层分发”的流程，将碎片化的 RSS 信息转化为高质量的结构化数据。

## 2. 核心架构：动静分离 (Decoupled Design)

项目采用 **“双分支协作”** 的现代 Serverless 架构，彻底解耦了代码逻辑与生成资产。

*   **Logic Layer (master)**：仅存放源码（Vue, Crawler）与 CI/CD 配置。保持洁净，仅在功能迭代时提交。
*   **Asset Layer (gh-pages)**：由 Actions 自动化管理，存放预编译网页、JSON 数据、Markdown 详情页及 RSS 订阅源。它是 Vercel 的真实部署源。

## 3. 设计哲学：数字纸张 (Digital Book Philosophy)

Alchemy 不仅仅是一个阅读器，它被设计为一本可以呼吸的“数字书籍”。

### 3.1 沉浸式视觉规范
*   **纸张质感**：采用微信读书风格的温润底色 (#EDF0F2) 与纯白书页容器，配合 24px 超大圆角，营造实体书的厚度感。
*   **磨砂玻璃拟态 (Glassmorphism)**：在 Header 和悬浮控制面板大量应用 `backdrop-filter: blur`，实现现代感与通透性的平衡。
*   **黄金排版比例**：详情页正文锁定 18px 字号、1.9 行高及 850px 最大宽度，旨在提供行业顶尖的长时间阅读舒适度。

### 3.2 交互式控制台
*   **全局状态共享**：通过 `useReadingSettings` 实现字号与主题的实时跨组件同步。
*   **物理反馈**：所有悬浮操作按钮均具备扩散阴影、上浮位移及缩放压感反馈。

## 4. 实现细节 (Implementation Details)

### 3.1 增量更新与去重
系统以 URL 的 MD5 哈希作为文章唯一指纹。通过对比存量 `links.json` 中的指纹，实现秒级的增量抓取，有效避免重复数据污染。

### 3.2 摘要提取与自动“脱水”
*   **提取**：从 RSS 源码中清洗 HTML 并截取前 200 字摘要。
*   **脱水策略**：为平衡 Git 仓库体积，仅在主索引中保留每个分类下最新的 100 条摘要。超出部分自动清除摘要字段，降级为标题与链接索引。

## 4. 未来演进与愿景 (10x Vision)

### 4.1 AI 数据炼金
*   **智能摘要**：集成 LLM 将长摘要精炼为 140 字“金句”。
*   **语义索引**：引入向量化（Embedding）技术，支持跨类别的语义搜索。

### 4.2 存储架构演进
针对摘要内容增长导致的 Git 瓶颈，系统规划了以下多级存储方案：
*   **数据分片 (Sharding)**：将历史数据按月度物理隔离，保持热数据索引的轻量化。
*   **混合存储 (Tencent COS)**：将索引留在 Git，而将海量的摘要/全文内容离线存储至腾讯云 COS，实现按需加载。
*   **内容寻址 (CAS)**：利用内容哈希实现全局去重，多源共用相同内容块。
