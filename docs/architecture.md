## 1. 系统概览
Alchemy-RSS 是一个高度自动化的**"技术知识档案馆"**。它通过集成 AI 增强的采集管道，将碎片化的 RSS 信息转化为高质量、永久存储的结构化数字资产。

## 2. 核心架构：混合数据源

项目采用 **"混合数据源 + 静态归档"** 架构，支持多种内容来源。

### 2.1 数据源类型

| 类型 | 配置字段 | 特性 | 示例 |
|------|----------|------|------|
| **BestBlogs API** | `category`, `sortType`, `timeFilter` | AI 摘要、评分、全文归档 | 精选文章、软件编程 |
| **标准 RSS** | `type: "rss"`, `xmlUrl` | 外部博客、仅索引 | 宝玉的分享、阮一峰 |

### 2.3 API 限速策略

采用业界标准的**指数退避 + 抖动** (Exponential Backoff with Jitter)：

| 重试次数 | 基础延迟 | 抖动范围 | 总延迟 |
|---------|---------|---------|--------|
| 1 | 1s | 0-1s | 1-2s |
| 2 | 2s | 0-1s | 2-3s |
| 3 | 4s | 0-1s | 4-5s |

可重试错误类型：
- `请求过于频繁` (中文限速提示)
- `rate limit` / `Too Many` / `429`
- `5xx` 服务器错误

### 2.4 前端策略：静态优先 (Static First)
Alchemy Reader 实现了"毫秒级"的阅读体验：
*   **Hit**：如果文章已归档，前端直接从 CDN 加载静态 JSON，无需等待后端计算，实现秒开且支持离线阅读。
*   **Miss**：对于极少数未归档的新文章，优雅降级为调用 Vercel Serverless API (`/api/reader`) 进行实时抓取。

## 4. 配置说明

### 4.1 数据源配置 (`data/rss.json`)

```json
[
  {
    "title": "精选文章 (BestBlogs)",
    "category": "Artificial_Intelligence",
    "sortType": "score_desc",
    "timeFilter": "3d",
    "qualifiedFilter": "true",
    "limit": 50
  },
  {
    "title": "宝玉的分享",
    "type": "rss",
    "xmlUrl": "https://baoyu.io/feed.xml",
    "limit": 20
  }
]
```

### 4.2 环境变量

| 变量 | 用途 | 必需 |
|------|------|------|
| `BESTBLOGS_API_KEY` | BestBlogs API 密钥 | 爬虫必需 |
| `GITHUB_TOKEN` | GitHub API 访问令牌 | 可选 |
| `WORKFLOW` | CI 模式标志 | 自动设置 |

## 5. 设计哲学：数字纸张 (Digital Book Philosophy)

Alchemy 不仅仅是一个阅读器，它被设计为一本可以呼吸的"数字书籍"。

### 5.1 沉浸式视觉规范
*   **纸张质感**：采用微信读书风格的温润底色 (#EDF0F2) 与纯白书页容器，配合 24px 超大圆角。
*   **Google 实用主义**：高对比度排版 (#202124 / #5F6368)，高密度列表视图，强调信息获取效率。
*   **磨砂玻璃拟态**：Header 和悬浮面板应用 `backdrop-filter: blur`，实现现代感与通透性的平衡。

### 5.2 智能交互
*   **AI Insight Card**：文章头部自动展示由 AI 提炼的"一句话摘要"和"核心观点"，帮助读者快速判断价值。
*   **Smart Categories**：摒弃传统的"按源分类"，采用基于内容的"智能标签聚类" (AI, Engineering, Product)，实现主题式阅读。

## 6. 存储架构 (The Archive Structure)

为解决 Git 仓库体积膨胀问题，我们实施了严格的分片策略：

```text
data/
├── rss.json            # 数据源配置
├── links.json          # 热数据索引 (含 AI 摘要、标签、归档路径)
└── articles/           # 冷数据归档 (全量内容，仅 API 源)
    ├── 2026/
    │   ├── 01/
    │   │   ├── a1b2c3d4.json
    │   │   └── ...
    │   └── 02/
    └── ...
```

*   **索引层**：轻量级，仅包含 metadata。
*   **内容层**：按年月物理分片，单个文件独立，Git 友好。
