# Alchemy-RSS: 架构分析与未来愿景 (Arch & Vision)

## 1. 项目现状 (Current State)

Alchemy-RSS 是一个基于 **Serverless** 理念构建的高度自动化的信息提炼管道。它将 RSS/Atom 订阅源转化为高质量的结构化数据（JSON, Atom）和人类/LLM 可读的 Markdown 文档。

### 1.1 核心特点
*   **配置驱动**：通过 `data/rss.json` 和环境变量灵活管理订阅源。
*   **异步并发**：利用 Node.js 异步特性高效处理多源抓取。
*   **增量更新**：通过链接指纹对比，确保仅处理新内容，避免重复。
*   **文档即数据**：生成的 Markdown 文件既是展示界面，也是 LLM 训练的高价值语料。

## 2. 架构深度解析 (Architectural Analysis)

项目采用了 **"Git-as-a-Database"** 和 **"Actions-as-a-Backend"** 的纯静态/Serverless 模式。

### 2.1 优势 (Strengths)
*   **零成本运营**：完全利用 GitHub 的基础设施（Actions, Pages, Storage）。
*   **天然的审计追踪**：每一次数据更新都有 Git 提交记录，方便回溯。
*   **极高的可用性**：依靠 GitHub 的稳定性，无需担心自建服务器宕机。
*   **对 AI 友好**：纯文本、结构化的输出非常适合作为 RAG（检索增强生成）的冷启动知识库。

### 2.2 局限 (Weaknesses & Risks)
*   **可扩展性瓶颈**：随着历史数据增加，Git 仓库体积会膨胀，导致 Actions 执行变慢。
*   **写入冲突**：由于是基于 Git 的推送，多实例并发写入时会产生冲突（虽然目前定时任务模式缓解了此问题）。
*   **网络访问受限**：GitHub Actions 的固定 IP 容易触发目标网站的反爬策略。
*   **交互性缺失**：作为静态系统，无法提供实时搜索、用户自定义过滤等动态交互。

## 3. 未来发展愿景: "AI 数据炼金术" (Future Vision)

结合 Google 的工程卓越理念与 Anthropic (Boris) 的 AI 实用主义，本项目可从“搬运工”向“洞察者”进化。

### 3.1 AI 提炼层 (The AI Layer)
*   **智能摘要**：集成 LLM（如 Gemini 1.5 Pro）自动为每篇文章生成 200 字摘要，极大提升信息获取效率。
*   **跨语言无障碍**：自动进行优质外文内容的中文翻译，打破信息茧房。
*   **信号与噪音评分**：利用 AI 根据内容深度进行打分，实现真正的“精选”。

### 3.2 架构演进 (Engineering Excellence)
*   **冷热数据分离**：将近期活跃数据留在 Git 仓库，历史归档转移至廉价的对象存储（如 R2/S3）。
*   **集成 MCP (Model Context Protocol)**：将项目改造为 MCP Server，使 LLM 能直接作为工具调用本库的实时数据。
*   **可观测性增强**：引入 SRE 监控理念，记录抓取成功率、耗时等指标，建立自动告警机制。

### 3.3 社区与扩展
*   **插件化解析器**：支持用户通过 WASM 或简单的脚本提交自定义网站的解析逻辑。
*   **去中心化分发**：利用分布式网络（如 IPFS）进一步增强数据的抗封锁能力。
